# Approval model

## Tier 0 — Observe

Read, import, validate, analyze, report, and create non-production tasks.

## Tier 1 — Investigate

Run diagnostics, fetch additional read-only detail, validate tracking, and create a research task.

## Tier 2 — Prepare

Create a draft, preview, diff, proposed campaign, content draft, code patch, or data migration plan. No production effect.

## Tier 3 — Execute reversible change

Apply a narrowly approved, reversible change with backup, bounded scope, audit log, verification, and rollback.

## Tier 4 — High-impact approval

Spend changes, publishing to a large audience, bulk contact, deletion, access or credential changes, irreversible migrations, policy-sensitive actions, and significant production modifications require a named human approver and enhanced verification.

An approval is bound to organization, project, provider, asset, action, scope, limit, executor, and expiry. It cannot be reused for materially different work.
